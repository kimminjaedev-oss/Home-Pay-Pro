Build a production-ready MVP property management payment web application using Next.js 15 (App Router), TypeScript, TailwindCSS, Prisma, PostgreSQL, and Stripe.

The application is for property/maintenance fee management for apartment unit owners (households).

Main Goal:
Users should be able to:

1. Register/login
2. Automatically see their unpaid balance from imported Excel data
3. Pay maintenance fees using Stripe
4. Pay custom amounts (partial payments allowed)
5. Admin should later manage payment status and households

The project should prioritize MVP speed and clean architecture.

========================

==========

Frontend:

* Next.js 15 App Router
* TypeScript
* TailwindCSS
* shadcn/ui components
* React Hook Form
* Zod validation

Backend:

* Next.js server actions / route handlers
* Prisma ORM
* PostgreSQL

Authentication:

* NextAuth/Auth.js
* Email/password authentication

Payments:

* Stripe Checkout
* Stripe Webhooks
* Support custom payment amounts

File Handling:

* Excel (.xlsx) upload parsing using SheetJS or similar library

Deployment-ready:

* Vercel compatible
* Environment variable support

========================
APP STRUCTURE
=============

Create these main pages:

1. Home Page

* Modern clean landing page
* Explain the service
* CTA buttons:

  * Login
  * Register
* Responsive design

2. Register Page
   Fields:

* Full name
* Email
* Password
* Unit number or household identifier

During registration:

* Match user against imported Excel data
* If household exists:

  * Automatically fetch unpaid balance
* Save unpaid balance into DB

3. Login Page

* Email/password login
* Secure authentication
* Redirect to dashboard

4. User Dashboard
   After login show:

* Household info
* Current unpaid balance
* Monthly fee ($36)
* Total suggested amount
* Payment history

Main feature:

* User can enter ANY payment amount manually
* Partial payment allowed
* Example:
  unpaid balance = $120
  monthly fee = $36
  user may pay:

  * full amount
  * partial amount
  * custom amount

Payment button:

* Stripe Checkout session

After payment:

* Save transaction history
* Update unpaid balance
* Show payment status

5. Admin Dashboard (basic MVP structure)
   Admin can:

* View all households
* View paid/unpaid users
* Search/filter users
* View payment history
* Export payment data to Excel

Admin routes should be protected.
